# Contributing to Fuel

## Pull Request Rules

- Every PR must link an issue
- Every PR must be reviewed by at least 1 person before merging
- Keep PRs small and focused — avoid large 2000-line PRs

## Issue Rules

Every issue must include:

- Clear description
- Acceptance criteria
- Labels
- Assignee
- Testing notes (screenshots, steps to reproduce, etc.)
- For bugs: steps to reproduce, expected behavior, and actual behavior

## Definition of Done

A task is only "done" when:

- Code is complete
- Tested locally
- PR reviewed and approved
- Linked PR merged
- No unresolved review comments
- No console/log errors
- Issue closed

## Code Quality Rules

- No commented-out code in PRs
- No hardcoded secrets
- No duplicate logic
- Keep functions small and focused
- Meaningful variable names
- No unused imports or variables
- Follow existing linting and formatting config

## Review Rules

Reviewers must check:

- Logic correctness
- Edge cases
- Code readability
- Naming clarity
- Basic security issues
- Respond or approve within 2 business days to avoid PRs stalling

## Communication Rules

| What | Where |
|---|---|
| Decisions | GitHub Discussions |
| Work | Issues |
| Code discussion | PR comments |
| Quick questions | GitHub Discussions |

No important decisions in chat only — document them in Discussions so they're searchable.
